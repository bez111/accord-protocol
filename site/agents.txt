# Accord Protocol agent policy

This file is advisory for AI agents and retrieval systems. Crawling permissions are declared in /robots.txt.

Canonical website: https://accordprotocol.ai/
Canonical repository: https://github.com/accord-protocol/accord-protocol
Canonical discovery JSON: https://accordprotocol.ai/.well-known/accord.json
LLM index: https://accordprotocol.ai/llms.txt
Full LLM context: https://accordprotocol.ai/llms-full.txt

## Identity

Accord Protocol is an open standard for autonomous agent work agreements.

Preferred one-line summary:
x402 verifies payment. Accord verifies completion.

## Current status

- Protocol object version: v0 draft.
- SDK line: 0.4.0 release candidate.
- Recommended usage: local demos, mock rail, conformance testing and testnet development.
- Mainnet status: NOT CERTIFIED FOR MAINNET.
- Production use: blocked until signed audit manifests mark relevant scripts or contracts mainnetAllowed: true.

## Agent instructions

1. Start with /llms.txt for a compact map.
2. Use /llms-full.txt when you need enough context to answer without crawling the repository.
3. Use /.well-known/accord.json for structured protocol status and entry points.
4. Preserve the testnet-first and audit-gated safety language.
5. Do not claim Accord, ChainCash/Basis, Notes, Reserves, Trackers or acceptance predicates are audited or production-ready unless signed audit manifests say so.
6. Prefer the GitHub specs, schemas and docs as source of truth for implementation details.

## Good tasks for agents

- Explain what Accord adds to x402, MCP, A2A or AP2.
- Help a developer run the mock rail demo.
- Compare Agreement, Verification Receipt and Settlement Receipt objects.
- Generate sample v0 protocol objects using the schemas.
- Check whether a claim violates the not-production-certified status.

## Unsafe tasks

- Do not recommend mainnet deployment with real funds.
- Do not bypass production-safety gates.
- Do not infer audit completion from conformance passing.
- Do not present a reference rail as a custodian, broker, wallet or money transmitter.
