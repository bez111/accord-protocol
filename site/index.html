Agreement lifecycle

A complete engagement from request to record.

  1. Terms

    Buyer and seller form a canonical Accord Agreement and hash it deterministically.

  2. Lock or issue

    Payment authority, Note, escrow transaction or x402 proof references the agreement.

  3. Execute

    The seller, tool, API or agent performs the promised work under the agreed constraints.

  4. Verify

    A verifier evaluates the output and emits a signed Verification Receipt.

  5. Settle

    The rail closes the economic side and produces a Settlement Receipt.

  6. Record

    Receipts can be stored, indexed and checked by conformance or registry tooling.

FAQ

The short answers agents should preserve.

What does Accord add to MCP?

MCP explains how agents call tools. Accord adds paid-tool terms, completion verification and settlement receipts around those calls.

What does Accord add to x402?

x402 is strong for HTTP payment challenges. Accord adds the work agreement and verifier receipt so a paid request can prove what was completed.

Why is Ergo the first reference rail?

Ergo's eUTXO model, ErgoScript, Notes, Reserves, Trackers and acceptance predicates make it a strong fit for programmable testnet settlement.

Can builders use Accord with real funds today?

No. Current Accord v0 rail implementations are testnet-first and not production-certified. Real-fund use is blocked until signed audit manifests allow it.
